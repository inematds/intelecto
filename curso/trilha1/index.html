🏪 Os 9 Frameworks de IA — Guia Completo

Conheça cada framework, sua linguagem, pontos fortes e para que tipo de projeto ele brilha. De TypeScript a Rust, de 4MB a 1GB de RAM.

O que é:

O framework mais completo — sistema operacional para IA pessoal. 10+ canais, 6+ provedores, 195K estrelas no GitHub. Tem voz, Canvas visual e apps mobile iOS/Android.

Por que aprender:

Se você quer a solução mais polida e completa disponível, com tudo funcionando junto — voz, canvas, multi-agente, plugins — OpenClaw é a referência.

Conceitos-chave:

Gateway Hub (WebSocket central), Plugin SDK, Wake Word, Canvas UI, 5.700+ skills disponíveis. RAM ~1GB. Melhor quando você quer o iPhone dos assistentes.

O que é:

OpenClaw reimaginado em Rust. Usa Trait Architecture — qualquer peça (provedor, canal, memória) é trocável sem tocar no resto. <5MB de RAM, boot em 10ms, 22+ provedores de IA.

Por que aprender:

Se eficiência e modularidade máximas são prioridade — especialmente em hardware limitado ou servidores com muitas instâncias — ZeroClaw é a escolha.

Conceitos-chave:

Traits (contratos), config.toml define tudo, ChaCha20 para criptografia. Curva de aprendizado alta (Rust), mas resultado é incomum em eficiência.

O que é:

Obra-prima de simplicidade: 5.200 linhas de código. Arquitetura de containers — cada grupo do WhatsApp roda isolado com seu próprio cérebro e CLAUDE.md.

Por que aprender:

Ideal para empresas com múltiplos grupos WhatsApp que precisam de isolamento total — sem vazamento de contexto entre clientes ou equipes.

Conceitos-chave:

Container isolation, Baileys (WhatsApp), CLAUDE.md por grupo, ~200MB RAM. Overkill para uso pessoal, perfeito para enterprise multi-grupo.

O que é:

Framework ultra-leve em Python com 4.000 linhas principais. Message Bus architecture — todos os canais comunicam por uma rodovia central. 9+ canais, 13+ provedores, 100MB RAM.

Por que aprender:

Você conhece Python e quer um framework enxuto que seja fácil de estender e entender linha a linha. Soul.md para personalidade, suporte a MCP.

Conceitos-chave:

Message Bus, Soul.md, 2 camadas de memória, suporte a email IMAP/SMTP. Bom equilíbrio entre simplicidade e funcionalidade.

O que é:

PicoClaw (Go): executa em hardware de $10, GPIO/I2C/SPI, <10MB RAM. IronClaw (Rust): segurança zero-trust com WASM sandbox e detecção de prompt injection. TinyClaw (TS): equipes de agentes que colaboram via fila de arquivos.

Por que aprender:

Cada um resolve um nicho específico: IoT/edge, compliance/segurança máxima, e pipelines de desenvolvimento com múltiplos agentes especializados.

Conceitos-chave:

PicoClaw = binário único Go + RISC-V. IronClaw = WASM sandbox + leak detection. TinyClaw = file-based queue + @coder, @reviewer, @writer colaborando.

O que é:

IA que fica mais inteligente com o uso. Cada solução encontrada é salva na memória — na próxima vez que a pergunta aparecer, o agente responde imediatamente. 20+ provedores, Docker, sub-agentes.

Por que aprender:

Para tarefas de pesquisa, análise exploratória e problemas em aberto onde flexibilidade > previsibilidade. Aprende com cada interação via SKILL.md.

Conceitos-chave:

Loop de aprendizado, memória de soluções, sub-agentes especializados, SKILL.md, Docker. Comportamento menos previsível — requer supervisão inicial.

🧰 A Mercearia — 130+ Funcionalidades Comparadas

Explore os 6 corredores de funcionalidades e o veredicto final — qual framework usar para cada caso de uso.

O que é:

7 funcionalidades que definem como seu AI se apresenta: Soul.md (personalidade em texto simples), AIEOS (padrão portátil), personas intercambiáveis, regras dinâmicas sem reinicialização.

Por que aprender:

Sem identidade bem definida, seu assistente é genérico e esquece regras. Com Soul.md você define tom, valores e comportamento de forma persistente.

Conceitos-chave:

SOUL.md → personalidade, AGENTS.md → regras, USER.md → preferências, MEMORY.md → memórias. Disponível em: OpenClaw, NanoBot, ZeroClaw, PicoClaw, NanoClaw, Agent Zero.

O que é:

WASM Sandbox (IronClaw), Container VM (NanoClaw), defesa contra prompt injection, detecção de vazamento de segredos, criptografia ChaCha20/Fernet, gates de aprovação em 3 níveis.

Por que aprender:

Se o assistente acessa dados sensíveis de clientes ou sistemas críticos, segurança não é opcional. Escolha o nível de proteção adequado ao seu risco.

Conceitos-chave:

Nível 1: somente leitura. Nível 2: supervisionado. Nível 3: autônomo. IronClaw tem a stack de segurança mais completa. ZeroClaw usa criptografia de nível militar.

O que é:

10 tipos de canal: WhatsApp, Telegram, Discord, Slack, Email, Painel Web, Voz (wake word), iMessage, SMS, Nostr. Cada framework suporta conjuntos diferentes.

Por que aprender:

Escolha o canal errado e seu assistente não estará onde seus usuários estão. Telegram é o mais simples de configurar — só precisa de um token de bot.

Conceitos-chave:

Telegram: 6 frameworks. WhatsApp: 6 frameworks. Discord: 5. Apenas OpenClaw tem voz por wake word. Todos implementam BaseChannel com start(), send(), stop().

O que é:

5 opções: busca híbrida (vetorial + keywords), SQLite FTS5 com BM25, sistema 2 camadas (MEMORY.md + histórico), memória de soluções (Agent Zero), base de conhecimento em documentos.

Por que aprender:

Memória determina custo (embedding APIs), precisão e capacidade de longo prazo. SQLite FTS5 cobre 90% dos casos sem custo de API.

Conceitos-chave:

~/.intelecto/memory.db persiste entre reinicializações. Categorias: fatos, resumos de conversas, soluções. Agent Zero tem o sistema mais avançado de aprendizado contínuo.

O que é:

7 capacidades: Cron (todos os frameworks), Heartbeat (verificações periódicas), automação de browser, webhooks (Zapier/n8n/Shopify), sub-agentes em background, times de agentes, jobs paralelos.

Por que aprender:

A diferença entre um chatbot e um assistente real é a capacidade de agir sem ser chamado. Agendamento cron transforma o assistente num executor proativo.

Conceitos-chave:

Cron: "toda segunda às 9h, envie briefing". Heartbeat: acorda a cada 30min para verificar tarefas. Webhooks: Shopify aciona o agente quando pedido chega.

O que é:

Mapa de decisão: Máxima completude → OpenClaw. Máxima eficiência → ZeroClaw ou PicoClaw. Entender cada linha → NanoClaw ou NanoBot. Segurança #1 → IronClaw. Times de agentes → TinyClaw ou Agent Zero.

Por que aprender:

Não existe framework melhor universal. Existe o certo para o seu caso. A mercearia deixa você montar sua lista de compras antes de construir.

Conceitos-chave:

WhatsApp Business com isolamento → NanoClaw. IoT e edge → PicoClaw. Python + fácil extensão → NanoBot ou Agent Zero. Máxima escolha de LLM → ZeroClaw (22+) ou Agent Zero (20+).
